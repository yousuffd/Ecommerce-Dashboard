# 📊 ECommerce Intelligence — C-Suite Dashboard

A **Streamlit-based executive analytics dashboard** for e-commerce operations, designed to give C-suite leaders a real-time, single-pane-of-glass view of revenue, operations, profitability, and customer health — powered by AI-generated insights via Claude (Anthropic) or ChatGPT (OpenAI).

[![Open in Streamlit](https://static.streamlit.io/badges/streamlit_badge_black_white.svg)](https://ecom-intelligence.streamlit.app)

---

## ✨ Features

### Dashboard Pages

**Executive Summary** — High-level KPI strip with health score, revenue momentum, fulfilment quality, and top-line business signals. Includes contextual alerts and AI-generated insights.

**Revenue** — Revenue trend analysis (monthly/weekly), MoM waterfall chart, category treemap, top products by revenue, and geographic breakdown.

**Operations** — Shipping performance (delivery bands, SLA breach rate, carrier analysis), order status distribution, return and cancellation trends.

**Profitability** — Gross margin analysis, discount impact, revenue concentration risk, and customer segment contribution.

**AI Insights** — Six board-ready, quantified insights generated by Claude or GPT-4o, covering growth momentum, operational risk, customer economics, and strategic recommendations.

**Ask the Data** — Conversational interface to query your dataset in natural language. Backed by Claude or GPT-4o with full awareness of your KPIs and dataset context.

| Tool | Model | Key Required |

| ChatGPT (OpenAI) | `gpt-4o` | `OPENAI_API_KEY` |

## 🚀 Getting Started

### Prerequisites

- Python 3.10+
- pip

### Installation

```bash
# 1. Clone the repository
git clone https://github.com/your-org/ecom-csuite-dashboard.git
cd ecom-csuite-dashboard

# 2. Install dependencies
pip install -r requirements.txt

# 3. Configure environment variables
cp .env.example .env
# Edit .env with your API keys and/or DB credentials

# 4. Run the app
streamlit run app_csuite.py
```

The app will open in your browser at `http://localhost:8501`.

---

## ⚙️ Configuration

Copy `.env.example` to `.env` and fill in only what you need:

```env
# AI keys (at least one recommended)
ANTHROPIC_API_KEY=sk-ant-api03-...
OPENAI_API_KEY=sk-...

# Database (optional — only if connecting to a live DB)
DB_TYPE=postgresql        # postgresql | mysql | sqlite
DB_HOST=localhost
DB_PORT=5432
DB_NAME=your_database
DB_USER=your_user
DB_PASSWORD=your_password
DB_SQLITE_PATH=            # path to .db file for SQLite
```

API keys can also be entered directly in the sidebar at runtime for single-session use. They are never stored or logged.

## 📋 Expected Data Schema

Your data file or database table should contain the following columns. Column names are normalised automatically (case-insensitive, spaces/hyphens converted to underscores).

### Required Columns

| Column | Type | Description |
|---|---|---|
| `order_id` | string | Unique order identifier |
| `order_date` | date | Order placement date |
| `product_name` | string | Product name |
| `category` | string | Product category |
| `quantity` | int | Units ordered |
| `unit_price` | float | Price per unit |
| `revenue` | float | Net order revenue |
| `city` | string | Customer city |
| `order_status` | string | e.g. Delivered, Shipped, Returned |
| `shipping_days` | int | Days from order to delivery |

### Optional Columns (enable richer metrics)

| Column | Type | Unlocks |
|---|---|---|
| `customer_id` | string | Repeat rate, LTV, unique customers |
| `customer_segment` | string | Segment-level analysis |
| `channel` | string | Acquisition channel breakdown |
| `sku_id` | string | SKU-level analysis |
| `discount_amount` | float | Discount rate, revenue lost to discounts |
| `cogs` | float | Gross margin % |
| `gross_margin` | float | Gross margin % (alternative to cogs) |
| `promised_delivery_date` | date | SLA breach detection |
| `actual_delivery_date` | date | SLA breach detection |
| `payment_method` | string | Payment mix analysis |
| `carrier` | string | Carrier performance breakdown |
| `return_reason` | string | Return reason analysis |

If `revenue` is absent but `unit_price` and `quantity` are present, revenue is derived automatically. Missing optional columns degrade gracefully — the relevant metrics are hidden rather than erroring.

## 🔒 Security Notes

- API keys entered in the sidebar are session-scoped and never persisted to disk or logs.
- Database credentials loaded from `.env` are read once per session.
- No data is sent to external services except the AI summary context (a plain-text KPI snapshot) when AI Insights or Ask the Data is used.
